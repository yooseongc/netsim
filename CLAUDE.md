# netsim

별도로 제공하는 **요구사항 문서**를 기준으로, 리눅스 패킷 경로 시뮬레이션 웹 서비스의 **실행 가능한 설계안**을 작성해주세요.

- 요구사항 정의: spec/REQUIREMENT.md
- 기술 사항: spec/TECH.md

설계 및 개발은 아래 문서를 마크다운으로 만들고 관리합니다. 항상 작업이 끝나면 문서를 업데이트하고, 작업이 시작하기 전에 문서를 읽습니다.

- docs/PROCESS.md : 진행 사항 정리
- docs/ISSUE.md : 개발 중 문제 사항 및 해결 방법을 정리
- docs/DESIGN_FRONTEND.md : 프론트엔드 설계안을 정리
- docs/STYLE.md : 프론트엔드 룩앤필, 공통 컴포넌트, 스타일 가이드를 정리
- docs/DESIGN_BACKEND.md : 백엔드 설계안을 정리
- docs/API.md : 백엔드 API를 정리한다.
- docs/DEV.md : 빌드 및 실행 방안 가이드
- docs/TEST.md : 테스트를 정의하고 현재 상태를 정리

## 유의사항

- 매 작업 시 이 문서를 읽고, 한 단위 작업 종료 시 각 문서를 업데이트 한다.
- 표준 스타일 가이드를 만들고 이를 준수한다. 필요 시 업데이트 할 수 있다.
- 시뮬레이션이 목적이므로 리눅스 네트워크 스택의 기본 동작을 시뮬레이션 하는데 충실한다. 
- 프론트엔드와 백엔드는 docs/API.md를 이용하여 상호간 계약을 성실히 이행한다.
- 프론트엔드는 페이지 전체의 look and feel을 일치하기 위해 노력한다.
- 코드를 책임 별로 명확히 분리, 작고 깔끔한 구조를 유지하기 위해 노력한다.
